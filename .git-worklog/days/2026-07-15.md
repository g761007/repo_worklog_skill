### 修復空儲存庫的分支名稱解析

- **異動內容：** 修復 collect_git_history.py 中的 repo_info() 函式，處理無提交記錄的儲存庫。原程式碼使用 git rev-parse --abbrev-ref HEAD，在空儲存庫上會失敗。改用 git symbolic-ref 正確解析未出生的分支。
- **程式碼行為：** Skill 現在可正確處理空儲存庫（無提交記錄），回報 has_commits:false 與正確的分支名稱，而非崩潰。

- **影響範圍：** Skill 更加健壯，可在各種初始化狀態的儲存庫上執行。
- **相關檔案：** `repo_worklog/scripts/collect_git_history.py`
